# 赛尔号原生桌宠

双击 `SeerPetDemo.app`，或执行：

```bash
open SeerPetDemo.app
```

- 按下左键立即播放受击动作，按住拖动时持续受击，松开立即恢复待机
- 桌宠窗口不受普通 macOS 窗口的屏幕边缘约束，可自由拖到屏幕顶部或跨出边缘
- 右键选择“调整大小”，可选 `0.25×`、`0.5×`、`0.75×`、`1×`、`1.25×`、`1.5×`、`1.75×`
- 右键可为当前精灵分别自定义四个动作名称；不同编号互不影响，也可一键重置
- 右键勾选“自由移动”后，精灵仅在待机时左右行走，碰到屏幕边缘折返；向左时待机和全部动作都会镜像
- 右键选择 `attack`、`sa`、`cp`、`hited`
- 右键选择“更换精灵…”，输入精灵编号即可下载、转换并换装
- 右键选择“退出桌宠”

应用内置 SWF 转换环境。首次切换某个编号时需要联网，完成后会缓存在本机；无需另装 Java、FFmpeg、Ruffle、WebView 或本地服务器。

首次转换精灵时，程序会分析官方战斗 SWF，寻找被 `attack`、`sa`、`cp` 三种动作共同引用的多帧身体动画，并将其中最内层的候选动画缓存为待机循环。这样不需要为每个编号手工配置 idle。少数旧精灵的 SWF 结构不兼容时，会显示 `sa` 的第一张可见站立帧并附加轻微平滑浮动，不会错误循环包含 attack、sa、cp、hited 的 `pet` 主容器。v0.6 生成的 30 帧错误待机缓存会自动删除。

同一动作的全部帧共用统一边界和缩放比例，避免大型特效造成逐帧漂移、缩放或出框。
播放大型动作时会自动扩大透明窗口，让精灵保持待机比例；待机和动作的第一帧按身体底部中心对齐，减少 300 号这类精灵切动作时的上下跳动。动作结束后恢复原尺寸和位置。特效特别大的精灵可能超出物理屏幕，但不会被桌宠窗口裁切。

内置待机帧的来源见 `THIRD_PARTY_ASSETS.md`。
