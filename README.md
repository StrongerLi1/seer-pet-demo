# 赛尔号原生桌宠

安装版：打开 `dist` 中的 DMG，把“赛尔号桌宠”拖入 Applications。

源码目录中也可以直接双击 `SeerPetDemo.app`，或执行：

```bash
open SeerPetDemo.app
```

重新编译、同步资源、测试并生成 DMG：

```bash
./package.sh
```

发布包使用 ad-hoc 签名；当前机器没有 Developer ID 证书，因此首次在其他 Mac 打开时可能需要在“系统设置 → 隐私与安全性”中确认。

## v1.45 更新

- 将精灵背包的“精灵恢复并居中”按钮改为“清理缓存”，面板会显示缓存路径、总占用和可清理空间，只删除当前未创建精灵的缓存
- 修复 9 号精灵待机动画显示不完整的问题，并保持其他精灵使用原有待机识别逻辑
- 为 70 号精灵屏蔽额外 `attack1` 对技能映射的影响，恢复“瞬雷天闪”的普通攻击动作和 17 帧动态待机；已有错误缓存会自动重建
- 增加 9、70、300 号精灵回归检查，防止特殊资源结构影响其他精灵

- 按下左键立即播放受击动作，按住拖动时持续受击，松开立即恢复待机
- 支持同时运行多个桌宠；每只桌宠都有独立名称、精灵编号、大小、自由移动、随机攻击和携带技能设置
- 常驻 macOS 菜单栏并从程序坞隐藏；点击菜单栏图标可打开宠物管理或退出程序
- 右键选择“宠物管理…”，可集中增加、编辑或删除桌宠；右键“重命名…”可直接修改当前桌宠名称
- 增加桌宠时只需输入精灵编号，名称默认使用官方编号对应名称；之后仍可单独重命名，编辑形象不会覆盖自定义名称
- 创建或编辑桌宠时可关闭“显示在桌面”；隐藏的宠物仍保留在管理列表，可随时重新显示
- “宠物管理”使用本地游戏 `ui2.swf` 提取的怀旧精灵背包：左侧直接使用原版 `126×73` 蓝色/橙色槽位、两列三行和选中帧，放大并下移显示精灵形象，名称、等级占位、HP 数值与血条整体向右下微调
- 背包槽位和右侧预览使用官方 `groupFightResource/pet/{编号}.swf` 中 `pet` 元件的正面首帧；程序优先读取本地游戏资源，不存在时再从官方地址获取，并为已有精灵自动补齐一次
- 不足 6 只精灵时，其余位置保留原版蓝色普通空槽，不显示精灵或文字
- 底部原版按钮保留悬停/按下效果，可控制跟随、首选、更换技能、随机攻击、增加、删除、编辑和清理缓存；清理面板会显示缓存路径和占用空间，并只清理当前未创建精灵的缓存；多于 6 只时复用右侧原版翻页箭头
- 管理窗口会裁掉原资源底部的透明黑边并移除右侧面板左上角的红色徽章；其余部分复用原版 `PetDataPanel` 面板、关闭按钮、技能槽、属性与性别图标素材。正面精灵图放大到 `100×120` 区域，名称栏按剩余空间自适应宽度，随后以相同间距排列属性和性别图标；技能槽按原版布局显示左上技能名、贴齐右上边界的属性图标、左下威力和右下 PP，属性攻击使用原版 `prop` 星形图标。字段恢复为序号、名字、等级、特性、升级经验、性格、获得时间、进化提示、六项战斗属性与六项学习力
- “获得时间”记录每个桌宠实例的创建日期并永久保存；六项属性按原版顺序以“攻击:数值”等形式显示官方 `PetXMLInfo` 基础值（攻击、防御、特攻、特防、速度、体力），桌宠没有的等级、培养值仍以 `--` / `0` 占位
- 四个原版技能槽显示官方携带技能、属性、威力和 PP；点击任一技能槽、底部技能按钮或桌宠右键“更换技能…”可从该精灵的原版已学技能中选择最多四个，配置按桌宠实例隔离保存
- 桌宠窗口不受普通 macOS 窗口的屏幕边缘约束，可自由拖到屏幕顶部或跨出边缘
- 右键选择“调整大小”，可选 `0.25×`、`0.5×`、`0.75×`、`1×`、`1.25×`、`1.5×`、`1.75×`
- 桌面精灵右键菜单直接显示当前配置的四个技能；物理攻击播放 `attack`，特殊攻击播放 `sa`，属性攻击播放 `cp`
- 右键勾选“自由移动”后，精灵仅在待机时左右行走，碰到屏幕边缘折返。程序会从官方地图精灵 SWF 自动提取 `left` / `right` 行走时间轴，并以 4 倍分辨率直接渲染 SWF 矢量帧；不兼容时才回退到待机移动。向左时战斗动作仍会同步镜像。
- 关闭自由移动时会保留最后朝向：向右停下后正常显示，向左停下后待机、受击和全部攻击继续镜像
- 右键可勾选“每 8 秒随机攻击”，自动从普通攻击、特殊攻击、属性攻击中选择一种；取消勾选后不再自动攻击
- 右键选择“更换精灵…”，输入精灵编号即可下载、转换并换装
- 右键选择“退出桌宠”

应用内置 SWF 转换环境。首次切换某个编号时需要联网，完成后会缓存在本机；无需另装 Java、FFmpeg、Ruffle、WebView 或本地服务器。

首次转换精灵时，程序会分析官方战斗 SWF，寻找被 `attack`、`sa`、`cp` 三种动作共同引用的多帧身体动画，并将其中最内层的候选动画缓存为待机循环。这样不需要为每个编号手工配置 idle。少数旧精灵的 SWF 结构不兼容时，会显示 `sa` 的第一张可见站立帧并附加轻微平滑浮动，不会错误循环包含 attack、sa、cp、hited 的 `pet` 主容器。v0.6 生成的 30 帧错误待机缓存会自动删除。

待机与行走会保留官方动画的完整帧率和清晰度。技能帧最高保留到 4096 像素，并在播放时只从磁盘加载当前一帧；程序同时限制转换器的 Java 堆内存，避免 5000 号等新精灵把数百张高清技能帧一次性装进内存。转换窗口会显示下载、分析、逐动作提取、移动动作和帧优化进度，旧缓存会自动迁移一次。

同一动作的全部帧共用统一边界和缩放比例，避免大型特效造成逐帧漂移、缩放或出框。
播放大型动作时会自动扩大透明窗口，让精灵保持待机比例；待机和动作的第一帧按身体底部中心对齐，减少 300 号这类精灵切动作时的上下跳动。动作结束后恢复原尺寸和位置。特效特别大的精灵可能超出物理屏幕，但不会被桌宠窗口裁切。

内置待机帧的来源见 `THIRD_PARTY_ASSETS.md`。
